Automatic Transaction Control

Using S2Tx.NET, it is possible to apply an Aspect to manage transaction to a .NET object. Before applying an aspect, refer to Warnings. Currently, s2dotnet uses MS DTC (Microsoft Distributed Transaction Coordinator) to manage transactions. Local transaction management is planned in future s2dotnet version. Following are transaction attributes available on S2Tx.NET:

Transaction Attributes

Followings Advices are defined in the Ex.dicon. Component name is the Advice name.

Attribute Component Name Description
Required Tx.RequiredTx If a transaction already exists then share it. Create a new transaction when necessary.
RequiresNew Tx.RequiresNewTx Regardless of the current status of a context, create a component by a new transaction.
Supported Tx.SupportedTx If a transaction exists, then share it.
NotSupported Tx.NotSupportedTx Create a component using transaction without a control context.

Example

IHoge.cs

namespace Examples.Tx
{
    public interface IHoge
    {
        void Foo();
    }
}

HogeImpl.cs

namespace Examples.Tx
{
    public class HogeImpl : IHoge
    {
        public void Foo()
        {
            Console.WriteLine("Foo");
        }
    
}

HogeClient.dicon

<components>
    <include path="Tx.dicon"/>
    <component class="Examples.Tx.HogeImpl">
        <aspect>Tx.RequiredTx</aspect>
    </component>
</components>

HogeClient.cs

using S2.NET.Framework.Container;
using S2.NET.Framework.Container.Factory;

namespace Examples.Tx
{
    public class HogeClient
    {
        private const string PATH =
            "Examples/Tx/HogeClient.dicon";

        [STAThread]
        static void Main(string[] args)
        {
            IS2Container container = S2ContainerFactory.Create(PATH);
            IHoge hoge = (IHoge) container.GetComponent(typeof(IHoge));
            hoge.Foo();
        }
    }
}

Result

DEBUG S2.NET.Extension.Tx.RequiredInterceptor - Transaction started
Foo
DEBUG S2.NET.Extension.Tx.RequiredInterceptor - Transaction committed

Tx.dicon is included in s2dotnet (under S2.NET project). Advice component name only needs to be specified as a value of an aspect tag.

Exception Handling

Advices defined in Tx.dicon will rollback a transaction when a component throws an exception.
