Introduction

S2Struts makes using Struts easier with Seasar. Service component is automatically set to Action class. Thus, Action class only has to call on the proper service component. Action class is strictly functions in a controller layer, so it can be separated from presentation layer and service layer. S2Struts, also, offer functionality to apply AOP to an Action.

Requirements

S2Struts requires the following software:
  • JDK1.4 or later
  • S2Container
  • S2Struts
Sample requires the following software. Please install and setup these software before trying to run the sample.
  • Tomcat
  • Tomcat Plugin

Setup

Follows the steps below to setup S2Struts to use on Eclipse:

  1. Extract files from the S2StrutsVx.x.x.jar distribution archive file.
  2. Open an Eclipse workspace and import the extracted files as a Java project. Answer "yes" to all prompts asking whether you want to overwrite the .classpath. Sample files are included in the src/examples folder.
Sample program and blank S2Struts template may be downloaded separately from S2Struts. To use these files, extract the downloaded archive files.
Follow the steps below to run the sample addition program:
  1. Create a Java project (not a Tomcat project) named s2struts-example
  2. Import the extracted sample files into this project. Answer "yes" to all prompts asking whether you want to overwrite the .classpath.
  3. Right click on the s2struts-example project and select "Property" -> "Tomcat"
  4. Restart Tomcat
  5. Open a web browser and open the following URI:
    http://localhost:8080/s2struts-example/

Usage

To start a S2Container, S2ContainerServlet is entered in web.xml.

web.xml

<web-app>
  <display-name>Struts Application</display-name>
  <servlet>
    <servlet-name>s2container</servlet-name>
   <servlet-class>org.seasar.framework.container.servlet.S2ContainerServlet</servlet-class>
    <load-on-startup/>
  </servlet>
  
  <!-- Standard Action Servlet Configuration (with debugging) -->
  <servlet>
    <servlet-name>action</servlet-name>
    <servlet-class>org.apache.struts.action.ActionServlet</servlet-class>
    <init-param>
      <param-name>config</param-name>
      <param-value>/WEB-INF/struts-config.xml</param-value>
    </init-param>
    <init-param>
      <param-name>debug</param-name>
      <param-value>2</param-value>
    </init-param>
    <init-param>
      <param-name>detail</param-name>
      <param-value>2</param-value>
    </init-param>
    <load-on-startup>2</load-on-startup>
  </servlet>

  <servlet-mapping>
    <servlet-name>s2container</servlet-name>
    <url-pattern>/s2container</url-pattern>
  </servlet-mapping>
 <!-- Standard Action Servlet Mapping -->

  <servlet-mapping>
    <servlet-name>action</servlet-name>
    <url-pattern>*.do</url-pattern>
  </servlet-mapping>
 <!-- The Usual Welcome File List -->

  <welcome-file-list>
    <welcome-file>index.jsp</welcome-file>
  </welcome-file-list>
 <!-- Struts Tag Library Descriptors -->

  <taglib>
    <taglib-uri>/tags/struts-bean</taglib-uri>
    <taglib-location>/WEB-INF/struts-bean.tld</taglib-location>
  </taglib>
  <taglib>
    <taglib-uri>/tags/struts-html</taglib-uri>
    <taglib-location>/WEB-INF/struts-html.tld</taglib-location>
  </taglib>
  <taglib>
    <taglib-uri>/tags/struts-logic</taglib-uri>
    <taglib-location>/WEB-INF/struts-logic.tld</taglib-location>
  </taglib>
  <taglib>
    <taglib-uri>/tags/struts-nested</taglib-uri>
    <taglib-location>/WEB-INF/struts-nested.tld</taglib-location>
  </taglib>
  <taglib>
    <taglib-uri>/tags/struts-tiles</taglib-uri>
    <taglib-location>/WEB-INF/struts-tiles.tld</taglib-location>
  </taglib>
</web-app>

To connect S2 with Struts, S2 RequestProcessor is entered in struts-config.xml. S2RequestProcessor and S2TilesRequestProcessor are available.

struts-config.xml

<struts-config>
  ...
  <controller
    processorClass="org.seasar.struts.S2RequestProcessor"/>
  ...
</struts-config>

It is necessary to register Action, which is to be used with S2, in a component definition. Furthermore, it is necessary to register this definition file with app.dicon, which holds the entire application definitions.

Add.dicon

<components>
    <component class="examples.struts.AddAction"/>
    <component class="examples.struts.AddServiceImpl"/>
</components>

app.dicon

<components>
    <include path="examples/struts/Add.dicon"/>
</components>

To receive service component, Action class needs to define a constructor or a setter method.

AddAction

package examples.struts;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import org.apache.struts.action.Action;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionForward;
import org.apache.struts.action.ActionMapping;

public class AddAction extends Action {

    private AddService addService_;

    public AddAction(AddService addService) {
        addService_ = addService;
    }

    public ActionForward execute(
        ActionMapping mapping,
        ActionForm form,
        HttpServletRequest request,
        HttpServletResponse response) {

        AddForm addForm = (AddForm) form;
        int result = addService_.add(addForm.getArg1(), addForm.getArg2());
        addForm.setResult(result);
        return (mapping.findForward("success"));
    }
}
